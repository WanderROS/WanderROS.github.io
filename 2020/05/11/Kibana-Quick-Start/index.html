Kibana Quick Start

发表于 2020-05-11 | 分类于 ELK

简介

Kibana是一种开源分析和可视化工具，可通过基于浏览器的界面轻松搜索，可视化和探索大量数据。借助Elasticsearch和Kibana，几乎可以浏览任何类型的数据，从文本文档到机器日志，应用程序指标，电子商务流量，传感器遥测或公司的业务KPI，一旦数据进入Elasticsearch，就可以在Kibana中进行探索并与之交互！可以使用Kibana搜索栏搜索数据，使用各种图表类型可视化数据，并使用实时仪表板进行可视化处理；还可以在大屏幕上显示仪表板，以提供整个公司或办公室的可见性。确保ELK的版本保持一致。 推荐书籍：《实战Elasticsearch、Logstash、Kibana——分布式大数据搜索与日志挖掘及可视化解决方案》· 高凯 编著

开始

安装并运行(7.6.2)

  1. macOS：

  2. docker:

  3. kibana默认的网址是：http://localhost:5601

  4. 配置：Kibana默认从$KIBANA_HOME/config/kibana.yml文件加载其配置

  5. 索引模式告诉Kibana您想探索哪些Elasticsearch索引，可以在Elasticsearch中为特定索引创建索引模式，也可以使用通配符*同时查询多个索引。在Kibana中可以有多个索引模式（就像数据库中有很多表一样）， 在创建可视化或搜索数据时，需要选择要在其上进行搜索的索引模式；

  6. 重要的导航栏：发现(Discover)，可视化(Visualize)和仪表板(Dashboard)

    1. 发现(Discover)：搜索和过滤原始文档的地方，每个记录都表示为一行，可以展开各行以查看每个记录中的所有字段及其值
    2. 可视化(Visualize)：可以使用许多现成的图表创建可视化并研究数据的工具，有两个核心定义值：
      • 存储桶聚合：存储桶聚合将文档分为多个存储桶，每个存储桶可以包含多个文档，一个文档或根本不包含任何文档
      • 指标聚合：创建存储桶后，指标聚合将为每个存储桶计算一个值
    3. 仪表板(Dashboard)：可以从多个角度查看数据并在同一视图中与数据进行交互
      • 选择图表的区域以放大特定的时间范围
      • 单击饼图中的一个切片以过滤该值

  7. 在Kibana的搜索中，有以下三种方式进行搜索：

    • KQL (Kibana Query Language)
    • Lucene
    • DSL：适用于使用Kibana中的Dev tools

  8. 可以使用 Kibana 来 search（查询），view（浏览）并且可以与存储在 Elasticsearch indices（索引）中的数据交互，可以很容易的以各种各样的 charts（图表），tables（表） 和 maps（地图） 样式来针对 data（数据）执行高级的 data analysis（数据分析）以及 visualize（ 可视化）；

  9. 基于浏览器的界面使得可以快速的创建并且实时显示修改的 Elasticsearch 查询的 dashboards（仪表盘）；

  10. Kibana 服务在启动时从 kibana.yml 文件中获取属性，默认设置将Kibana配置为在localhost:5601上运行。需要更改主机或端口号，需要更新kibana.yml文件，还可以启动SSL并设置各种其它的选项：

    • server.port :默认值 ，5601 Kibana，由后端服务器提供服务，此设置指定要使用的端口
    • server.host :默认值 ， “localhost” ，此设置指定后端服务器的主机
    • server.basePath :使您能够指定如果您在代理后运行的 Kibana 的路径
    • server.maxPayloadBytes :默认值 :，1048576 传入服务器请求的最大有效负载大小
    • server.name :默认值 “your-hostname” 用于标识此 Kibana 实例的可读的显示名称
    • server.defaultRoute :默认值 ，“/app/kibana” 此设置指定打开 Kibana 时的默认路由
    • elasticsearch.url :默认值 ，“http://localhost:9200” 要用于所有查询的 Elasticsearch 实例的 URL
    • elasticsearch.preserveHost :默认值，true
    • kibana.index :默认值，“.kibana”Kibana 使用 Elasticsearch 中的索引来存储保存的搜索
    • kibana.defaultAppId :默认值， “discover” 要加载的默认应用程序
    • tilemap.url :Kibana 用于在 tilemap 可视化中显示地图图块的图块服务的 URL
    • tilemap.options.minZoom :默认值，1 最小缩放级别
    • tilemap.options.maxZoom :默认值，10 最大缩放级别
    • server.ssl.cert 和 server.ssl.key :分别指向 PEM 格式 SSL 证书和 SSL 密钥文件的路径
    • elasticsearch.ssl.ca :可选设置，使您能够为 Elasticsearch 实例指定证书颁发机构的 PEM 文件的路径
    • elasticsearch.ssl.verify :默认值，true 要忽略 SSL 证书的有效性，请将此设置的值更改为 false
    • … …

  11. Kibana 支持客户端请求和 Kibana 服务器发送到 Elasticsearch 的请求的 SSL 加密；

  12. Elasticsearch不仅是开源分布式检索与数据处理平台，而且是一个基于Lucene构建的开源、分布式、RESTful信息检索框架；

  13. ELK架构为数据分布式存储、可视化查询和日志解析创建了一个功能强大的管道链；

  14. ELK对于大数据处理、信息检索以及搜索引擎研发、日志处理与分析、挖掘信息可视化等、对于设计高效的大型商业网站都具有重要的现实意义；

  15. Elasticsearch支持服务器集群，节点之间是对等关系的（去中心话），Master节点多了维护集群状态的功能，每个节点上的集群状态数据都是实时同步的，Master节点出故障，会选出一个节点成为Master节点；

  16. Elasticsearch API的简单使用：

    1. 通过HTTP方式的JSON格式进行调用
    2. 通过Thrift软件框架方式，Thrift结合了功能强大的软件堆栈和代码生成引擎，以构建在C++、Java等编程语言间的无缝、高效服务
    3. 通过Memcached方式，分布式内存对象缓存系统，用于在动态系统中减少数据库负载，提升性能
    4. 节点客户端，以一个无数据节点的身份加入到集群中
    5. 传输客户端，更加轻量的传输客户端用来项远程集群发送请求

  17. Elasticsearch集群中各个节点都是通过9300端口进行通信；

  18. Elasticsearch支持安装插件，可以下载插件然后在Elasticsearch的安装目录进行安装操作；

  19. 部分插件介绍：

    • Analysis-ansj——ansj中文分词器
    • Analysis-ik——IK中文分词器
    • Analysis-mmseg——mmseg中文分词
    • Analysis-pinyin——拼音分词器
    • Analysis-string2int——字符串转整形工具，可以用在Facets、Aggregations功能上
    • Bigdesk——监控Elasticsearch状态
    • Head——对Elasticsearch进行多种操作的客户端工具
    • Inquisitor——调试查询
    • Kopf——网络管理

  21. Elasticsearch可以接受本地、共享以及云平台上的数据，在Lucene提供的基本功能上，通过构建分布式索引，完成对大数据的加工处理；

  22. Elasticsearch 中建立索引的简要流程：

    1. 准备待处理的文档；
    2. 将准备好的数据提交文档给Elasticsearch；
    3. 完成索引；
    4. 返回索引结果。

文档索引及管理

  1. Elasticsearch中对文档的索引等进行操作时，既可以通过RESTful接口进行操作，也可以通过Java客户端进行操作；

  2. 文本数据首先要被加工处理，即建立索引文件之后才能被检索到；

  3. 建立索引方法有：

    • 倒排索引（最常用）
    • 后缀数组索引
    • 签名文件索引
    • … …

  4. 一个关闭的索引将禁止写入或读取其中数据的操作，而一个打开的索引文件可以允许用户对其中的数据文件进行操作；

  5. 相对于Lucene的索引，Elasticsearch索引过程多了分布式数据的扩展，主要用Tranlog进行各个节点间的数据平衡；

  6. Elasticsearch提供多种途径对文档进行诸如获取信息、增删该、更新等相关操作；

  7. 实现人机间自然语言的交互，意味着要使用计算机既能理解自然语言的意义，也能以自然语言文本来表达给定的意图、思想等。前者是自然语言处理（NLP，Natural Language Processing）,后者是自然语言生成；

  8. 在进行自然语言处理时，特别是对像中文这样的语言来说分词是非常必要的；

信息检索及结果过滤

  1. 在Elasticsearch中的RESTful接口方式中，完成信息检索功能的关键词是_search，通过POST方式发送到Elasticsearch，其后再跟“?q=查询词”等；
  2. 信息检索时，可简单地使用基于Lucene的通用检索语法，也可以使用灵活的基于JSON格式的Query DSL（Domain Special Language）来构造各种检索请求；
  3. 在处理字段、类型和查询时可以指定分析器；
  4. Elasticsearch在执行带有filter的查询时，会打开索引的每个segment段文件，然后去判断里面的文档是否符合filter要求，并且这个匹配的结果用一个很大的只有两个状态的数组BitSet来存储;

信息统计分析与搜索提示

  1. 在面向关系型数据库管理系统中的SQL语句中，可以使用集函数完成简单的统计分析工作；
  2. 基于facets可以得到各种统计结果（可以是以数组形式返回单的）；
  3. terms facets能够指定一个字段进行统计，Elasticsearch将返回在指定字段中使用最多的词项及其分布；
  4. range facets可以允许指定区间并在这个区间内进行统计并获得指定字段的统计数据；
  5. histogram facets是一种可以根据其返回值生成可以将来用于柱状图的统计数据；
  6. date_histogram facets是一个增强型的专门针对于日期型字段进行统计的facets；
  7. statistical facets 可以对数字类型的字段进行统计；
  8. terms_stats facets是terms facets和statistical facets的组合形式，提供了在一个字段上基于另一个字段获得的取值进行统计的能力；
  9. aggregations是facets的升级版，新版本中将废弃facets功能，相应的功能转而由aggregations替代；

Elasticsearch配置

  1. Elasticsearch.yml负责设置服务器的默认状态：
    1. 集群名称 cluster.name: “ELK”
    2. 节点名称 node.name: “Franz Fafka”
    3. 节点是否为master节点 node.master: true
    4. 节点是否存储数据 node.data: true
    5. 设置绑定的IP地址 network.bind_host
    6. 设置节点和其他节点交互的TCP端口，默认是9300 transport.tcp.port
    7. … …
  2. 集群中的索引可能由多个分片构成，且每个分片可能拥有多个副本，通过将一个单独的索引分成多个分片，可以处理由于文件太大而不能在一个单一机器上运行的大型索引；
  3. 通过监控集群状态，可以使管理员了解集群的整体运行情况，这对及时发现可能存在的问题是很有必要的；

Kibana可视化

  1. 信息可视化技术是利用计算机实现对抽象数据的可视表示,用来增强人们对抽象信息的感知;
  2. 河流模型是将得到的海量信息资源集合,按照时间维度构造一个类似河流的可视化显示方式;
  3. 关联分析模型是按照新闻报道的要素,通过提取并分析时间、地点、分类等属性，构造关联分析模型，用于发现它们之间可能存在的关联关系；
  4. Kibana是由类似栅格的仪表盘构成的，而仪表盘是由行和面板组成的；
  5. 如果在本地启动Kibana，则可以使用浏览器打开http://localhost:5601;
