Elasticsearch Quick Start

发表于 2020-05-07 | 分类于 ELK

简介

如何使用切面技术（faceting）基于查询结果来计算汇总数据，如何使用新引进的聚合框架（分析引擎，可以为你的数据赋予意义）。学习如何使用Elasticsearch的空间搜索能力（spatial capability）和预搜索（prospective search）使得我们的数据更有用是做这篇文章的主要目的！GitHub使用Elasticsearch搜索20TB的数据！推荐书籍：《Elasticsearch 服务器开发（第二版）》· [波兰] Rafa Kuc Marek Rogozinskl著 蔡建斌译

开始

基础知识

  1. Elasticsearch：优秀的全文检索和分析引擎

  2. 文档（document）：索引和搜索时使用的主要数据载体，包含一个或多个存有数据的字段

  3. 字段（field）：文档的一部分，包含名称和值两部分

  4. 词（term）：一个搜索单元，表示文本中的一个词

  5. 标记（token）：表示在字段文本中出现的词，由这个词的文本、开始和结束偏移量以及类型组成

  6. 倒排索引建立索引中词和文档之间的映射，不同于以往的正排索引

  7. 分析的工作由分析器完成，它由一个分词器（tokenizer）和零个或多个标记过滤器（token filter）组成，也可以有零个或多个字符映射器（character mapper）

    • 小写过滤器（lowercase filter）：把所有的标记变成小写
    • 同义词过滤器（synonyms filter）：基于基本的同义词规则，把一个标记换成另一个同义的标记
    • 多语言词干提取过滤器（multiple language stemming filter）：减少标记（实际上是标记中的文本部分），得到词根或者基本形式，即词干

  8. 通过使用多个过滤器，几乎可以达到无限的分析可能性

  9. 评分（scoring）：根据文档和查询的匹配度用计分公式计算的结果，Apache Lucene使用TF/IDF（term frequency/inverse document frequency，词频/逆向文档频率）评分机制（一种计算文档在我们查询上下文中相关度的算法）

  10. 索引（index）是Elasticsearch对逻辑数据的逻辑存储，可以分为更小的部分，可以把索引存放在一台机器或者分散在多台服务器上，每个索引有一或多个分片（shard），每个分片可以有多个副本（replica）

  11. 存储在Elasticsearch中的主要实体叫文档（document），文档由多个字段组成， 每个字段可能多次出现在一个文档里， 这样的字段叫多值字段（multivalued）

  12. 在Elasticsearch中，一个索引对象可以存储很多不同用途的对象

  13. Elasticsearch在映射中存储有关字段的信息，每一个文档类型都有自己的映射，即使我们没有明确定义

  14. CRUD（create-retrieve-update-delete，增删改查）

核心概念

  1. Elasticsearch可以运行在许多互相合作的服务器上， 这些服务器称为集群（cluster），形成集群的每个服务器称为节点（node）
  2. 数据可以分为较小的称为分片（shard）的部分，每个分片可以放在不同的服务器上，数据可以在集群的节点中传播，多个分片可以加快索引
  3. 副本（replica）只是一个分片的精确复制，每个分片可以有零个或多个副本，用于提高查询吞吐量或实现高可用性
  4. Elasticsearch处理许多节点，集群的状态由时光之门控制
  5. Elasticsearch REST API可用于各种任务，可以管理索引，更改实例参数，检查节点和群集状态，索引数据，搜索数据或者通过GET API检索文档
  6. 查询信息（本地搭建）：
    1. 基本信息：http://localhost:9200/
    2. 集群中节点的信息：http://localhost:9200/_cluster/state/nodes/
    3. 关闭节点：http://localhost:9200/_cluster/nodes/_shutdown

索引

  1. 更多分片和副本：
    • 更多分片使索引能传送到更多服务器，意味着可以处理更多文件，而不会降低性能
    • 更多分片意味着获取特定文档所需的资源量会减少，因为相较于部署更少分片时，存储在单个分片中的文件数量更少
    • 更多分片意味着搜索索引时会面临更多问题，因为必须从更多分片中合并结果，使得查询的聚合阶段需要更多资源
    • 更多副本会增强集群系统的容错性，因为当原始分片不可用时，其副本将替代原始分片发挥作用
    • 更多副本意味着查询吞吐量将会增加，因为执行查询可以使用分片或分片的任一副本
  2. 模式映射（schema mapping，或简称映射）用于定义索引结构:
    • 唯一标识符
    • 名称
    • 发布日期
    • 内容
  3. 每个字段类型可以指定为Elasticsearch提供的一个特定核心类型，核心类型有：
    • string：字符串
    • number：数字
    • date：日期
    • boolean：布尔型
    • binary：二进制
  4. 数值字段类型有：
    • byte：定义字节值
    • short：定义短整型值
    • integer：定义整型值
    • long：定义长整型值
    • float：定义浮点值
    • double：定义双精度值
  5. 开箱即用的分析器：
    • standard：方便大多数欧洲语言的标准分析器
    • simple：这个分析器基于非字母字符来分离所提供的值，并将其转换为小写形式
    • whitespace：这个分析器基于空格字符来分离所提供的值
    • stop：这个分析器类似于simple分析器，但除了simple分析器的功能，它还能基于所提供的停用词过滤数据
    • keyword ：这是一个非常简单的分析器， 只传入提供的值
    • pattern：使用正则表达式灵活地分离文本
    • language：旨在特定的语言环境下工作
    • snowball：提供了词干提取算法
  6. 可用的相似度模型：
    • Okapi BM25模型：基于概率模型，概率模型估算根据指定查询找到指定文档的概率
    • 随机性偏差（divergence from randomness）模型：这种相似度模型基于具有相同名称的概率模型
    • 信息基础（information-based）模型：这是新推出的最后一个相似度模型，与随机性偏差模型非常相似
  7. 合并调度器指示Elasticsearch合并过程的方式，有两种可能：
    • 并发合并调度器：默认，在独立的线程中执行，定义好的线程数量可以并行合并
    • 串行合并调度器：在调用线程（即执行索引的线程）中执行。合并进程会一直阻塞线程直到合并完成
  8. Elasticsearch计算文档标识符的散列值，以此为基础将文档放置于一个可用的主分片上，接着，这些文档被重新分配至副本
  9. 路由可以控制文档和查询转发的目的分片

搜索

  1. 无论想执行的操作是更改映射还是文档索引化，都要用REST API向Elasticsearch发送JSON结构的数据；

  2. Elasticsearch支持两种类型的查询：基本查询和复合查询；

  3. 查询Elasticsearch最简单的办法是使用URI请求查询；

  4. Elasticsearch能控制想要的最多结果数以及想从哪个结果开始：

    • from，该属性指定我们希望在结果中返回的起始文档，默认值是0
    • size，该属性指定了一次查询中返回的最大文档数，默认值为10

  5. Elasticsearch提供一项功能，让我们可以根据文档需要满足的最低得分值，来过滤结果；

  6. Elasticsearch是一个分布式搜索引擎，因此提供的所有功能在性质上都必须是分布式的；

  7. Elasticsearch允许通过指定搜索类型来选择查询在内部如何处理，为了控制查询如何执行，可以使用search_type请求参数，并将其设置为下列值之一：

    • query_then_fetch
    • query_and_fetch
    • dfs_query_and_fetch
    • dfs_query_then_fetch
    • count
    • scan

  8. Elasticsearch具有广泛的搜索和数据分析能力，以不同的查询、筛选和聚合等形式公开；

  9. 多词条查询允许匹配那些在内容中含有某些词条的文档。词条查询允许匹配单个未经分析的词条，多词条查询可以用来匹配多个这样的词条；

  10. match查询把query参数中的值拿出来，加以分析，然后构建相应的查询；

  11. Elasticsearch支持下列专用过滤器：

    • bool过滤器
    • geo_shape过滤器
    • has_child过滤器
    • has_parent过滤器
    • ids过滤器
    • indices过滤器
    • match_all过滤器
    • nested过滤器
    • prefix过滤器
    • range过滤器
    • regexp过滤器
    • term过滤器
    • terms过滤器

  12. Elasticsearch在内部为每个过滤器都建立了一个叫bitset的结构，它保存着索引中的后续文档是否跟过滤器匹配的信息 ，Elasticsearch允许我们控制高亮片段的数量以及它们的大小，为此公开了两个属性供使用 ；

扩展索引结构&更好的搜索

  1. 为索引数组，只需要在数组名称中指定字段的属性；
  2. 得分是描述文档与查询相关度的一个参数；
  3. Lucene返回文档时，意味着文档与我们发送的查询匹配，并且对该文档已给出一个分数；

超越全文检索

  1. 可以使用Elasticsearch作为各种系统的一个关键部分，处理大量的数据，提取结论，并将这些数据可视化为可读的方式；
  2. 有两组聚合：度量聚合（ metric aggregation）和桶聚合（ bucketing aggregation）
    • 度量聚合接收一个输入文档集并生成至少一个统计值
    • 桶聚合返回很多子集，并限定输入数据到一个特殊的叫做桶的子集中
  3. 查询是最简单的切面类型，它允许在切面结果中得到与查询匹配的文档数量；
  4. 除了使用查询以外， Elasticsearch允许使用过滤器来计算切面；
  5. 切面允许我们指定一个字段， Elasticsearch使用这个字段并返回该字段上最频繁的词条；
  6. 基于范围的切面使我们可以根据定义好的一组范围获取文档数，此外，还可以获取指定字段的汇总数据，每个范围
    返回以下统计信息：
    • from：定义了范围的左界（如果在查询中指定）
    • to：定义了范围的右界（如果在查询中指定）
    • min：定义了给定范围内，用作切面的字段的最小值
    • max：定义了给定范围内，用作切面的字段的最大值
    • count：定义了落在指定范围内的字段值的文档数目
    • total_count：定义了落在指定范围内的字段值的文档总数
    • total：定义了落在指定范围内的所有字段值的总和
    • mean：定义了落在指定范围内用作range切面的字段值的均值
  7. 直方图切面允许你根据字段值的间隔来建立一个字段值的直方图；
  8. statistical切面允许我们为数值型字段计算统计数据，在返回数据中，我们得到计数、总和、平方和、平均值、最小值、最大值、方差和标准偏差等统计信息；
  9. 查询中包含的过滤器并不会缩小切面结果的范围，切面的计算时基于与你的查询匹配的文档；
  10. Elasticsearch提供了以下三种建议器的类型：
    • term，这种建议器更正每个传入的单词，在非短语查询中很有用，比如单词条查询
    • phrase，这种建议器工作在短语上，返回一个恰当的短语
    • completion，这种建议器旨在提供快速高效的自动完成结果
  11. Elasticsearch中，多词条过滤器乍看是个非常简单的过滤器！

深入集群

  1. 启动一个Elasticsearch节点时，该节点会开始寻找具有相同集群名字并且可见的主节点。如果找到主节点，该节点加入一个已经组成了的集群；
  2. 默认情况下， Elasticsearch允许节点同时成为主节点和数据节点；
  3. ping是一个节点间发送的信号，用来检测它们是否还在运行以及可以响应。主节点会ping集群中的其他节点，其他节点也会ping主节点；
  4. Elasticsearch允许配置何时启动最初的恢复过程，恢复是初始化所有分片和副本的过程，从事务日志中读取所有数据，并应用到分片上；
  5. 字段数据缓存是Elasticsearch缓存的一部分，主要用于当查询对字段执行排序或切面时；
  6. 字段数据断路器（ field data circuit breaker）允许估计一个字段加载到缓存所需的内存；
  7. 一个好的软件可以管理硬件故障或人为错误等异常情况；
  8. 快照保存它创建的时间点上所有跟集群相关的数据，包括集群状态和索引的信息，至少在创建第一个快照之前，必须创建一个快照存储库；
  9. Elasticsearch提供了非常详细的信息，使你能够检查和监控单个节点或作为一个整体的集群；
  10. 一个最基本的API是集群健康度API，它允许使用单个HTTP命令得到整个集群的状态信息；
  11. Elasticsearch索引是保存数据的地方，它对大多数部署来说是非常重要的部分；
  12. Elasticsearch提供了下列信息：
    • merges：该节点包含Lucene段合并的信息
    • refresh：该节点包含刷新操作的信息
    • flush：该节点包含清理信息
    • warmer：该节点包含预热器的信息，以及它们执行了多久
    • filter_cache：这些是过滤器缓存统计信息
    • id_cache：这些是标识符缓存统计信息
    • fielddata：这些是字段数据缓存统计信息
    • percolate：该节点包含预匹配器使用情况的信息
    • completion：该节点包含自动完成建议器的信息
    • segments：该节点包含Lucene段的信息
    • translog：该节点包含事务日志计数和大小的信息
  13. 节点统计API跟前面描述的节点信息API类似。主要的区别是，节点信息API提供环境信息，而节点统计API告诉我们集群工作时发生过什么；
  14. 再平衡是在集群的不同节点之间移动分片的过程；
  15. Elasticsearch有两种主要类型的插件。这两种类型可以基于其内容来分类： Java插件和站点插件（ site plugins），Elasticsearch把站点插件当成被内置的HTTP服务器处理的文件集，于/_plugin/plugin_name/URL（如/_plugin/bigdesk）下面；
  16. Elasticsearch允许在elasticsearch.yml文件中指定各种参数来调优。但应该把这个文件当做默认设置，可以在运行时通过Elasticsearch REST API修改！

总结

Elasticsearch是基于Lucene的新一代分布式、RESTful风格的开源搜索引擎，具有实时搜索、稳定、快速、安装使用方便等优点。学习ELK技术对于自己在数据上的分析非常的有帮助！
