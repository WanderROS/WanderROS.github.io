JavaScript 基础强化

发表于 2020-06-17 | 分类于 JavaScript

简介

JavaScript 是一种可以用来给网页增加交互性的编程语言。“站在其他程序员肩膀上”的方式正是熟悉 JavaScript 的好方法！Java 已经成为一种在服务器端编写代码的流行语言，而JavaScript也在浏览器端名声大噪。 推荐书籍：《JavaScript基础教程（第9版）》· [美] Dori·Smith Tom·Negrino 著 柳靖 等 译

开始

能干什么

  1. 用 JavaScript 可以做许多事情，使网页更具交互性，给站点的用户提供更好、更令人兴奋的体验；
  2. 使用 JavaScript 来确保用户以表单形式输入有效的信息，这可以节省业务时间和开支；
  3. 使用 JavaScript，根据用户的操作可以创建自定义的 Web 页面；
  4. JavaScript 还可以处理表单，设置 cookie，即时构建 HTML 页面以及创建基于 Web 的应用程序！

基础知识

  1. 用来表示文档中对象的标准模型就称为DOM（Document Object Model，文档对象模型）；

  2. 树中的每个对象也称为树的节点（node）；

  3. 事件（event）是用户在访问页面时执行的操作，提交表单和在图像上移动鼠标就是两种事件；

  4. JavaScript 使用称为事件处理程序（event handler）的命令来处理事件：

    • 在编写脚本时，不必预测出用户可能采取的所有操作，只需处理那些你希望提供特殊处理的事件
    • 如果没有 onload 事件处理程序，页面也会顺利地加载

  5. CSS 是一种用于 Web 的标准布局语言，可以控制版面、颜色以及元素和图像的大小和位置；

  6. 优秀的站点应该分离,应该包含三种文件:

    • HTML：包含页面的内容和结构
    • CSS：控制页面的外观和表现
    • JavaScript：控制页面的行为

  7. CSS 样式表在规则中使用class 和 id属性定义页面的外观，而 JavaScript 文件在代码中使用class 和 id属性来影响页面上元素的行为:

    • 类（class）标识出可能会多次使用的元素,用于定位多个元素

    • id 标识出的元素对于文档是唯一的,用于定位指定元素

    • 在给定的页面上，一个 id 只能出现一次。“1”是一个数字，而井号（#）也称为数字符，所以这个符号用于 id

  8. 养成在脚本中添加注释的习惯是一种很好的做法，事实上不管代码写的多优秀，时间一久还是容易不记得，经验告诉我们还是要有正确的注释，不是那种无用的注释，比如注释i++为自加操作！注释有助于开发者和其他开发人员理解代码；

  9. 编写任何脚本都有许多种方式，并且都能实现同样的效果！相信自己写的代码是正确的，多做单元测试！

  10. 在 JavaScript 代码中，要想避免与变量作用域有关的问题和混淆，最容易的方法是避免使用同名的两个变量在不同的地方做不同的事。如果必须使用同名的变量，就一定要弄清变量的作用域；

  11. JavaScript 关键字 this 使脚本能够根据使用这个关键字的上下文将值传递给函数；

  12. 使用 JavaScript 的 try/throw/catch 命令可以产生友好、有用的错误消息；

  3. 对于检查浏览器支持哪些对象，另一种替代方法是进行浏览器探测（browser detection），这种方法尝试查明用户使用哪种浏览器查看页面；

  4. 数组是一种可以存储一组信息的变量，数组可以包含任何类型的数据：文本字符串、数字、其他 JavaScript 对象,数组编号是从零开始的：

窗口&表单

  1. 在Web 浏览器中，窗口是最重要的接口元素，JavaScript 提供了很多用来操纵窗口的工具；

  2. 在 JavaScript 中，窗口形成一个层次结构，父窗口处于这个层次结构的顶层；

  3. HTML 页面调用可以打开新窗口的外部 JavaScript 文件：

  4. 如果需要从 Web 站点的用户那里收集信息，那么就需要使用表单；

  5. 表单可以包含大多数常见的图形界面元素，包括输入字段、单选按钮、复选框、弹出菜单和输入列表；

  6. JavaScript 是检查数据的好方法，这种技术称为表单验证（form validation），在客户机上用 JavaScript 进行验证要快得多，而且用户操作的效率也更高；

  7. 可以使用 JavaScript 检查某些或所有字段是否已经填写了，指定用户必须填写某些字段，然后才能提交表单；

  8. 将输入字段的边框改为红色是很不错的，但是如果能够让有问题的字段再醒目一点儿就更好了；

  9. 正则表达式（regular expression，常常缩写为 RegExp）是一种用特殊符号编写的模式，描述一个或多个文本字符串；

  10. 只要把混乱的正则表达式分解成有意义的小块，其语法并不难理解；

工具包、框架&库

  1. jQuery 库的优点：
    • 轻量级
    • 活跃的开发社区
    • 插件架构
    • 支持旧浏览器
    • 速度
    • 初级开发人员容易掌握
  2. 在网站的页面中使用jQuery库，引入库就可以了；
  3. 在基于 jQuery 的 JavaScript 代码中，人们首先注意到的通常是美元符号$。$符号仅仅表示一个合法但不常见的函数名，jQuery 通过它来获取所需的一切；
  4. 即使访客的浏览器端没有缓存所需的库文件，CDN（Content Delivery Network，内容发布网络）的因特网连接性也要优于你自己实现的网站，所以 CDN 提供文件的速度会优于你的服务器；
  5. jQuery 流行的原因之一是它的选择器与 CSS 中的选择器非常相似，这样一来，偏重于设计的开发人员就可以更快地掌握它！
  6. jQuery本身可能不支持某一功能，但是可以添加插件来获取功能，比如排序可以使用插件tablesorter；
  7. jQuery 通过 jQuery UI 来处理常用的用户界面元素；
  8. ThemeRoller 允许用户根据项目需要来定制 jQuery 的用户界面主题；
  9. 开发人员为了扩展 jQuery 的核心功能而创建了各式各样的插件；

事件

  1. 当浏览器探测到一个事件时，比如用鼠标单击或按键，它可以触发与这个事件相关联的 JavaScript 对象，这些对象称为事件处理程序（event handler）;
  2. 事件处理实际上包含了用户与页面的所有交互；
  3. 当用户执行某些会影响整个浏览器窗口的操作时，就会发生窗口事件；
  4. 当用户进入页面而且所有页面元素都完成加载时，就会触发onload事件；
  5. 当用户离开网页时，就会触发 onunload 事件处理程序，onbeforeunload在用户开始离开页面之前触发，而 onunload 在用户离开页面之后触发；
  6. 事件：
    • 当窗口调整大小时，会触发 onresize 事件处理程序
    • 当窗口移动时，会触发 onmove 事件处理程序
    • 当用户取消网页上的图像加载时，会触发 onabort 事件处理程序
    • 当页面上发生 JavaScript 错误时，可能会触发 onerror 事件处理程序
    • 当一个页面成为最前面的活动窗口时，就会触发 onfocus 处理程序；而当一个页面退回后台的时候，就会触发 onblur 事件处理程序
    • 当用户向上或者向下滚动页面时，就会触发 onscroll 事件
    • onmouseup 事件会在用户单击鼠标然后释放按钮时触发
    • 当页面的访问者移动鼠标时，就会触发 onmousemove 事件
    • 当鼠标移动进任何注册了 onmouseover 事件处理程序的区域时，就会触发这个事件
    • 当鼠标离开一个注册了此事件的区域时，就会触发onmouseout事件
    • onclick 要求用户按下鼠标按钮并放开才能触发单击事件
  7. 键盘事件：
    • 通过使用键事件处理程序，可以在用户按下适当的键时执行相应的操作，比如onkeydown事件
    • onkeyup 事件处理程序在用户已按下并释放键的过程中触发
    • 当用户按下并释放键时触发 onkeypress 事件

cookie

  1. cookie 是一小段信息，当用户第一次访问 Web 服务器时，服务器将这些信息发送给浏览器；

  2. 浏览器将 cookie（其中包含关于访问者的信息）作为纯文本文件保存在计算机硬盘上；

  3. cookie 技术背后的本质是“识别”，cookie 是一个具有特定格式的文本字符串：

  4. 可以使用 cookie 和 JavaScript 提醒经常访问站点的用户注意他们没看到过的内容;
