NoSQL Qucik Start

发表于 2020-06-25 | 分类于 NoSQL

简介

NoSQL和高性能、分布式、海量数据等热词有着千丝万缕的联系！随着用户内容的增长，所生成、处理、分析和归档的数据的规模快速增大，类型也快速增多，这些大数据集通常被称为大数据。 *推荐书籍：《深入NoSQL》· [印] Shashank Tiwari著 巨成 译 *

开始

基础知识

  1. NoSQL并不单指一个产品或一种技术，代表一族产品，以及一系列不同的、有时相互关联的、有关数据存储以及处理的概念；
  2. 金无足赤，SQL虽好，但是面对大量稀疏数据时就很鸡肋，这个时候NoSQL就发挥作用了！
  3. NoSQL缓解了RDBMS引发的问题并降低了处理海量稀疏数据的难度，但是反过来也被夺去了事务完整性的力量和灵活的索引以及查询能力；
  4. 可扩展性：
    • 配置一个大而强的资源来满足额外的需求
    • 依靠普通机器组成的集群
  5. 每个数据单元可以看作一组键/值对集合，单元本身通过主标识符（primary identifier）标识，主标识符又叫主键；
  6. 每条数据中只有合法的键/值对才会被存储；
  7. 哈希表或关联数组是可以容纳键值对的最简数据结构；
  8. EHCache是一个开源分布式缓存系统广泛应用于各类Java应用中；
  9. Memcached在Web应用中非常流行 ，是开源的高性能对象缓存系统；
  10. 文档数据库中的文档一词意指文档中松散结构的键值对集合，通常是JSON,而非一般意义的文档或表格；
  11. 当今为数不多的开源文档数据库中，声明远扬的数MongoDB和CouchDB；
  12. 图形数据库和XML数据存储，也可以算作NoSQL数据库，图形数据库有：Neo4j和FlockDB！

大数据产品

HBase

  1. 在线资源：http://hbase.apache.org/
  2. 历史：创建于Powerset，微软收购Powerset前HBase被捐赠给了Apache基金会
  3. 技术和语言：Java实现
  4. 访问方法：JRuby的shell支持命令行访问，有Thrift、Avro、REST和protobuf客户端
  5. 查询语言：无原生查询语言，由Hive提供类SQL接口
  6. 开源许可证：Apache License版本2
  7. 使用者：Facebook、StumbleUpon、Hulu、Ning、Mahalo、Yahoo！等

Hypertable

  1. 在线资源：http://www.hypertable.org
  2. 历史：创建于Zvents，现在是独立开源项目
  3. 技术和语言：C++实现，使用Google RE2正则表达式类库
  4. 访问方法：支持命令行，支持Thrift接口
  5. 查询语言：HQL（Hypertable Query Language， Hypertable查询语言）是一种用于查询Hypertable数据的类SQL抽象
  6. 开源许可证：GNU GPL版本2
  7. 使用者：Zvents、百度和Rediff

Cloudata

  1. 在线资源：http://www.cloudata.org
  2. 历史：由韩国开发者YK Kwon创建，没有太多有关其起源的公开信息
  3. 技术和语言：Java实现
  4. 访问方法：支持命令行，支持Thrift、REST和Java API
  5. 查询语言：CQL（Cloudata Query Language， Cloudata 查询语言）是一种用于查询Cloudata数据的类SQL抽象
  6. 开源许可证：Apache License版本2
  7. 使用者：不详

  7. MongoDB中，集合是弱结构的，因此可以在同一个集合中允许存储不同的文档类型；

NoSQL区分

  1. NoSQL分类：

    • 文档存储——MonogoDB和CouchDB
    • 键值存储——Redis和BerkeleyDB
    • 列族存储——HBase和Hypertable
    • 最终一致的键值存储——Apache Cassandra和Voldermot

  2. 作为文档存储的一种，MongoDB可以存储任意数据集合，只要数据可以用JSON式的对象层次结构表示就可以；

  3. MongoDB支持的不仅仅是JSON，还有BSON（二进制编码序列化JSON结构）；

  4. Redis是持久化键值存储，为了保持高效，它把数据库放在内存里，然后利用异步线程将其写入磁盘；

  5. HBase和列数据库按列存储数据，空值不需要存储，因此非常有利于保存稀疏数据集；

  6. Apache Thrift是开源的跨语言的服务开发框架，是代码生成引擎，用来创建能够与多种编程语言交互的服务，使用步骤如下：

    1. 下载Thrift
    2. 解压源码
    3. 构建和安装Thrift
    4. 编写Thrift服务定义，这是最重要的部分，用来生成代码的底层定义
    5. 使用Thrift编译器生成特定语言的代码

  8. HBase使数据的结构扁平化,只有列族和它包含的列之间才有层次关系；

  9. MonogoDB查询文档是数据类型敏感的；

  10. MapReduce是Google申请了专利的软件框架，支持在大型分布式集群上进行分布式计算；

  11. Google和MonogoDB的MapReduce功能也从函数式编程世界的类似结构中得到了启发；

  12. MapReduce可以帮助我们在分布式的数据集合上执行大规模计算！

Hive&大数据

  1. Apache Hive是建立在Hadoop基础上的数据仓库，Apache Pig是大数据分析的高级语言！
  2. Hive在Hadoop上提供了优雅的类SQL查询框架，有很好的扩展性，能处理海量的分布式数据集，但是不是实时查询系统；
  3. Hive中不存在查询缓存,因此重复查询耗时和第一次查询一样!
  4. Hive支持非常全面的过滤和聚合查询，可以用WHERE子句来过滤数据，也可以使用GROUP BY分组；
  5. Hive支持内连接、外连接和左外连接；
  6. Hive基于Hadoop并利用MapReduce,Hive里的计划解释可以揭示查询背后的MapReduce！
  7. 表分区可以将数据分到多个命名空间中去，并基于命名空间标识符对数据进行过滤和查询；
  8. Hive开源并作为Hadoop的一个子项目，现在已经是顶级项目，而且继续处于快速发展中，填补着SQL和NoSQL之间的空白！

总结

主要介绍的还是MonogoDB！NoSQL泛指非关系型数据库，之前看过书籍介绍Redis，这部分学习了MonogODB以及了解了NoSQL相关的基础知识，大数据相关的知识也有所介绍，但是远远不够！之前听过一本书名《左手MonogoDB右手Redis》，现在是摸着两种技术的边了，后续可以继续深入学习！
