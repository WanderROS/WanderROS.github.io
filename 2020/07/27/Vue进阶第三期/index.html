简介

针对第二期过程中对于Vue路由管理过程中出现的不解，进行系统化地学习与实操从而更深入理解Vue。在这部分也记录了Element-ui中的自定义图标以及回到顶部没有出现效果的解决办法。这部分没有华丽的特效，但是对于前端开发项目很有帮助！

开始

Vue-Router

  1. Vue Router 是 Vue.js 官方的路由管理器，和 Vue.js 的核心深度集成，让构建单页面应用变得易如反掌；

  2. 功能：

    • 嵌套的路由/视图表
    • 模块化的、基于组件的路由配置
    • 路由参数、查询、通配符
    • 基于 Vue.js 过渡系统的视图过渡效果
    • 细粒度的导航控制
    • 带有自动激活的 CSS class 的链接
    • HTML5 历史模式或 hash 模式，在 IE9 中自动降级
    • 自定义的滚动条行为

  3. 当把 Vue Router 添加进项目中进行路由管理时，需要做的是将组件 (components) 映射到路由 (routes)，然后告诉 Vue Router 在哪里渲染它们；

  4. 使用Vue-router管理路由器后，就可以在任何组件内通过 this.$router 访问路由器，也可以通过 this.$route 访问当前路由；

  5. 可以在创建 Router 实例的时候，在 routes 配置中给某个路由设置名称：

  6. 通过对router-view命名可以在界面中拥有多个单独命名的视图，而不是只有一个单独的出口，这样可以同时 (同级) 展示多个视图，而不是嵌套展示，也有可能使用命名视图创建嵌套视图的复杂布局；

  7. 动态路由匹配：

    1. 需求：业务上经常需要将匹配某种模式的路由，全部映射到同一个组件上；

    2. 方法：在 vue-router 的路由路径中使用“动态路径参数”(dynamic segment) 来满足这种需求；

    3. 案例：有一个 User 组件，对于所有 ID 各不相同的用户，都要使用这个组件来渲染；

    4. 样例：

    5. 一个“路径参数”使用冒号 : 标记，当匹配到一个路由时，参数值会被设置到 this.$route.params，然后可以在组件内使用；

    6. 在一个路由中设置多段“路径参数”，对应的值都会设置到 $route.params 中：

      模式 匹配路径 $route.params
      /user/:username /user/evan { username: 'evan' }
      /user/:username/post/:post_id /user/evan/post/123 { username: 'evan', post_id: '123' }

    7. 当使用同一个组件来进行渲染时，原来的组件实例会被复用，而不会被销毁，从而导致组件的生命周期钩子不会被再调用，复用时想要对路由参数的变化做出响应，可以通过watch监测route对象；

    8. 匹配任意路径，我们可以使用通配符 (*)；

  8. 嵌套路由：

    1. 需求：实际开发中，一个前端页面通常由多层嵌套的组件组合而成；

    2. 方法：URL 中各段动态路径也按某种结构对应嵌套的各层组件

    3. 一个被渲染组件可以包含自己的嵌套 router-view，即组件可以嵌套router-view；

    1. 编程模型：

    2. 在 Vue 实例内部，可以通过 $router 访问路由实例，因此可以调用 this.$router.push；

    3. 方法的参数示例：

      • 如果提供了 path，params 会被忽略
      • onComplete，onAbort回调将会在导航成功完成 (在所有的异步钩子被解析之后) 或终止 (导航到相同的路由、或在当前导航完成之前导航到另一个不同的路由) 的时候进行相应的调用

  10. 路由组件传参：

    1. 可以通过$route进行参数传递，但是在组件中使用$route 会使之与其对应路由形成高度耦合，从而使组件只能在某些特定的 URL 上使用，限制了其灵活性；
    2. 使用 props 将组件和路由解耦，提升组件灵活性；

  11. vue-router模式：

    • 使用 URL 的 hash 来模拟一个完整的 URL，于是当 URL 改变时，页面不会重新加载，即hash 模式（默认）， hash 值的变化，并不会导致浏览器向服务器发出请求，浏览器不发出请求，也就不会刷新页面。另外每次 hash 值的变化，还会触发hashchange 这个事件，通过这个事件我们就可以知道 hash 值发生了哪些变化

    • URL 就像正常的 url，例如 http://yoursite.com/user/id，即 history 模式

  12. 导航守卫：

    1. vue-router 提供的导航守卫主要用来通过跳转或取消的方式守卫导航；

    2. 参数或查询的改变并不会触发进入/离开的导航守卫，可以通过观察 $route 对象来应对这些变化；

    3. 全局前置守卫：

    4. 可以在路由配置上直接定义 beforeEnter 守卫；

    5. 可以在路由组件内直接定义以下路由导航守卫：

      • beforeRouteEnter
      • beforeRouteUpdate
      • beforeRouteLeave

  13. 动态过渡：

    1. 使用transition为组件给添加一些过渡效果；
    2. 想让各个路由组件有各自的过渡效果，可以在各路由组件内使用 <transition> 并设置不同的 name；
    3. 可以基于当前路由与目标路由的变化关系，动态设置过渡效果；

Vue数据获取

  1. 进入某个路由后，有时候需要从服务器获取数据。例如，在渲染用户信息时，需要从服务器获取用户的数据；
  2. 获取阶段：
    • 导航完成之后获取：先完成导航，然后在接下来的组件生命周期钩子中获取数据。在数据获取期间显示“加载中”之类的指示
    • 导航完成之前获取：导航完成前，在路由进入的守卫中获取数据，在数据获取成功后执行导航

导航完成后获取数据

  1. 使用这种方式时，会马上导航和渲染组件,然后在组件的 created 钩子中获取数据；

  2. 有可能需要在数据获取期间展示一个 loading 状态，或者在不同视图间展示不同的 loading 状态；
