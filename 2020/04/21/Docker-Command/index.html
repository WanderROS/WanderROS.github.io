Docker Command

发表于 2020-04-21 | 分类于 Docker

简介

Docker想要学好的话，首先要学会使用它，Docker命令非常的多，这篇文章中主要介绍一些日常经常用到的命令，熟练使用Docker命令，然后能够部署Docker容器，最后在这个基础上深入Docker的核，从长远的角度来说，Docker技术会引导行业变革,产业升级!

开始

run命令

  1. -a，–attach：把指定的数据流（如 STDOUT 之类）连接至终端；
  2. -d，–detach：使容器在“分离”模式下运行，容器会在后台运行，而命令的返回值是容器的ID；
  3. -i, –interactive：保持 stdin 打开，与-t同时使用，用作交互式会话容器；比如：docker run -it mysql /bin/bash
  4. –restart：配置 Docker 在什么情况下尝试重新启动已退出的容器；
  5. –rm：退出时自动删除容器，不能与 -d 选项同时使用；
  6. -t, –tty：分配一个伪终端（pseudo-TTY）；
  7. -e, –env：设置容器内的环境变量，–env-file 选项可以经文件传入环境变量；
  8. -h, –hostname NAME：设置容器的unix主机名为 NAME；
  9. –name NAME：把 NAME 设置为容器的名称；
  10. -v, –volume：用来设置数据卷；
  11. –volumes-from：挂载指定容器拥有的数据卷；
  12. –expose：指定容器将会使用的端口或端口范围，只有与 -P 参数同时使用，以及在连接容器时，才有真正意义；
  13. –link：建立一个与指定容器连接的内部网络接口；
  14. -p, –publish：“发布”容器的端口，使主机能访问它；
  15. -P, –publish-all：“发布”所有已指定为开放（exposed）的容器端口，使主机能访问它们，可通过 docker port 命令查看分配了哪个端口；
  16. –entrypoint：把参数指定为容器的入口（entrypoint），覆盖任何Dockerﬁle中的ENTRYPOINT指令；
  17. -u, –user：设置命令运行时所使用的用户，覆盖任何Dockerﬁle中的USER指令；
  18. -w, –workdir：将参数的路径设置为容器的工作目录。此选项会覆盖 Dockerﬁle 中的 WORKDIR 指令。

容器管理命令

  1. docker attach [OPTIONS] CONTAINER：attach 命令允许用户查看容器内的主进程，或与它进行交互；
  2. docker create ：从镜像创建容器，但不启动它；
  3. docker cp：在容器和主机之间复制文件和目录；
  4. docker exec：在容器中运行一个命令，可用于执行维护工作，或替代 SSH 用作登入容器；
  5. docker kill：发送信号给容器中的主进程（PID 1）；
  6. docker pause：暂停容器内的所有进程；
  7. docker restart：重新启动一个或多个容器；
  8. docker rm：删除一个或多个容器；
  9. docker start：启动一个或多个已停止的容器；
  10. docker stop：停止（但不删除）一个或多个容器；
  11. docker unpause：重启先前被 docker pause 命令暂停的容器。

Docker信息命令

  1. docker info：打印Docker系统和主机的各种信息；
  2. docker help：把一个子命令作为参数，打印有关该子命令的使用方法和帮助信息；
  3. docker version：打印Docker客户端和服务器版本，以及编译时使用的Go版本。

Docker容器信息命令

  1. docker diff：对比容器所使用的镜像，显示容器的文件系统的变化；
  2. docker events：打印守护进程的实时事件；
  3. docker inspect：把容器或镜像作为参数，获取它们的详细信息；
  4. docker logs：输出容器的“日志”；
  5. docker port：把容器作为参数，列出它的端口映射信息；
  6. docker ps：提供关于当前容器的高阶信息， 例如名称、ID 和状态；
  7. docker top：把容器作为参数， 提供该容器内运行中进程的信息。

Docker镜像管理命令

  1. docker build：从Dockerﬁle建立镜像；
  2. docker commit：从指定的容器创建镜像；
  3. docker export：将容器的文件系统中的内容以 tar 归档的格式导出，并输出到 STDOUT；
  4. docker history：输出镜像中每个镜像层的信息；
  5. docker images：列出所有本地镜像，包括库名称、标签名称以及镜像大小等信息；
  6. docker import：从一个含有文件系统的归档文件创建镜像，归档可以由 docker export 产生；
  7. docker load：加载仓库， 仓库以 tar 归档的形式从 STDIN 读入；
  8. docker rmi：删除指定的一个或多个镜像；
  9. docker save：把指定的镜像或仓库储存到 tar 归档， 并输出到 STDOUT ；
  10. docker tag：将镜像与一个仓库和标签名称关联。

Docker仓库命令

  1. docker login：在指定的寄存服务器进行注册或登录；
  2. docker logout：从Docker寄存服务器注销；
  3. docker pull：从寄存服务器下载指定的镜像；
  4. docker push：将镜像或仓库推送到寄存服务器；
  5. docker search：列出 Docker Hub 上匹配搜索词的公共仓库。

总结

这部分是常用的Docker命令,可以进行速查,加深一下印象吧!
