Linux Platform

发表于 2020-04-27 | 分类于 Linux

简介

Linux是一个很好的平台，因为它是一个透明的系统,特别是大多数系统配置都存放在文本文件中，让人一目了然!Linux这样的现代操作系统非常复杂，内部有多得令人眼花缭乱的各种组件在同步运行和相互通信。学会使用抽象思维来思考问题！

开始

基本概念

  1. Linux操作系统主要分为三层，最底层是硬件系统，包括内存和中央处理器等；硬件系统之上是内核，它是操作系统的核心，内核管理硬件系统，是硬件系统和应用程序之间进行通信的接口；进程是指计算机中运行的所有程序，由内核统一管理，它们组成了最顶层，称为用户空间；
  2. 内核在内核模式（kernel mode）中运行，而用户进程则在用户模式（user mode）中运行；
  3. 只有内核可以访问的空间我们称为内核空间（kernel space）；用户空间指的是那些用户进程能够访问的内存空间；
  4. 内核负责管理：
  • 进程：内核决定哪个进程可以使用CPU
  • 内存：内核管理所有的内存，为进程分配内存，管理进程间的共享内存以及空闲内存
  • 设备驱动程序：作为硬件系统（如磁盘）和进程之间的接口，内核负责操控硬件设备
  • 系统调用和支持：进程通常使用系统调用和内核进行通信
  1. 一个进程让出CPU使用权给另一个进程称为上下文切换（context switch）；
  2. 内核是在上下文切换时的时间段间隙中运行的；
  3. 系统调用（system call或syscall）为进程执行一些它们不擅长或无法完成的工作，比如打开、读取和写文件这些操作都涉及系统调用；
  4. 虚拟设备对于用户进程而言是物理设备，但是通过软件来实现的；
  5. 用户机制主要用于权限管理,用户组是指一组用户的集合。用户组的主要作用是允许一个用户同组内的其他用户共享文件权限;

基础命令

  1. cat：显示一个或者多个文件的内容；
  2. ls：显示指定目录的内容，默认参数为当前目录；
  3. cp：用来复制文件；
  4. mv：用来重命名或者移动文件；
  5. touch：用来创建文件，如果文件已经存在，则该命令会更新文件的时间戳；
  6. rm：删除文件，文件一旦被删除通常无法恢复；
  7. echo：将它的参数显示到标准输出；
  8. cd：用来设置当前工作目录；
  9. mkdir：用来创建新目录；
  10. rmdir：用来删除目录，使用rm -rf来删除一个目录以及其中的所有内容；
  11. pwd：输出当前的工作目录名；
  12. diff：用来查看两个文件之间的不同；
  13. file：查看一个文件的格式信息；
  14. find，locate：查找文件；
  15. sort：将文件内的所有行按照字母顺序快速排序；
  16. man：获取系统的帮助；
  17. >&：将标准输出和标准错误输出重定向到同一个地方；
  1. <：操作符将文件内容重定向为命令的标准输入；
  2. ps：列出所有正在运行的进程；
  3. kill：发送一个信号给指定的进程；
  4. 任务控制命令：
    1. fg：将进程置于前台；
    2. bg：将进程移入后台；
    3. jobs：查看暂停了哪些进程；
    4. &：将进程设置为后台运行；
  5. chmod：更改文件权限；
  6. ln -s：创建符号链接；
  7. sudo：以root用户身份执行命令；

Linux设备管理

  1. 在Unix系统中操纵大多数设备都很容易，因为很多I/O接口都是以文件的形式由内核呈现给用户的；
  2. 设备类型：
  • 块设备：按固定的块大小读取数据
  • 字符设备：只能对字符设备读取和写入字符数据
  • 管道设备：输入输出端不是内核驱动程序，而是另外一个进程
  • 套接字设备：跨进程通信经常用到的特殊接口
  • 终端设备：在用户进程和输入输出设备之间传送字符
  1. dd命令是从输入文件和输入流读取数据然后写入输出文件和输出流，在此过程中可能涉及到编码转换；
  2. 小型计算机系统接口（Small Computer System Inteface，SCSI）；
  3. mknod命令用来创建设备，必须知道设备名以及主要和次要编号，可以用于在系统恢复的时候创建丢失的设备文件；

文件系统

  1. 分区是对整块磁盘的进一步划分,内核将分区用块设备呈现，如同每个分区是一整块的磁盘,分区数据存放在磁盘上的分区表中;

  2. 文件系统是用户空间中与你日常交互的文件和目录数据库;

  3. 分区表：

    1. 主引导记录（Master Boot Record，MBR）
    2. 全局唯一标识符分区表（Globally Unique Identifier Partition Table，GPT）

  4. 虚拟文件系统（Virtual File System，以下简称VFS）负责文件系统的具体实现；

  5. 第四扩展文件系统（ext4）：支持更大的文件和更多的子目录个数；

  6. mkfs能够自己决定设备上的块数量并且设置适当的默认值，用来创建文件系统；

  7. 系统可以通过使用磁盘空间来扩展内存容量，如果出现内存空间不足的情况，Linux虚拟内存系统会自动将内存中的进程移出至磁盘以及从磁盘移入内存，用来保存内存页面的磁盘空间我们称为交换空间；

  8. 创建交换分区：

    1. 确保分区为空
    2. 运行mkswap dev，其中dev是分区设备
    3. 运行swapon dev向内核注册
    4. 在/etc/fstab文件中创建一个新的交换条目，这样系统在重启之后即可使用该交换空间

  9. 如果不想重新分区或者不想新建交换分区的话，你可以使用常规文件作为交换空间：

    1. 创建一个空文件：dd if=/dev/zero of=swap_file bs=1024k count=num_mb
    2. 将其初始化为交换空间：mkswap swap_file
    3. 将其加入交换池：swapon swap_file

  10. 可以使用swapoff命令来删除交换分区或交换文件；

  11. 高性能网络服务器需要尽可能避免磁盘存取和交换空间；

  12. 传统的Unix文件系统有两个基础组件：一个用来存储数据的数据块池和一个用来管理数据池的数据库系统；

内核启动

  1. 简化流程：
    1. BIOS或者启动固件加载并运行引导装载程序
    2. 引导装载程序在磁盘上找到内核映像，将其载入内存并启动
    3. 内核初始化设备及其驱动程序
    4. 内核挂载root文件系统
    5. 内核使用PID 1来运行一个叫init的程序，用户空间在此时开始启动
    6. init启动其他的系统进程
    7. init还会启动一个进程，通常发生在整个过程的尾声，负责用户登录
  2. 在启动时，Linux内核的初始化过程:
    1. 检查CPU
    2. 检查内存
    3. 检测设备总线
    4. 检测设备
    5. 设置附加内核子系统（如网络等）
    6. 挂载root目录
    7. 启动用户空间
  3. 运行Linux内核的时候，引导装载程序会向内核传递一系列文本形式的内核参数来设定内核的启动方式；
  4. 几乎所有的磁盘设备都由固件系统供BIOS通过线性块寻址（Linear Block Addressing）来访问硬件，引导装载程序往往是唯一使用BIOS访问磁盘的程序；
  5. GRUB工作原理：
    1. BIOS或者固件初始化硬件，在启动存储设备上寻找启动代码
    2. BIOS和固件运行找到的启动代码，开始GRUB
    3. 加载GRUB核心
    4. 初始化GRUB核心，此时GRUB可以读取磁盘和文件系统
    5. GRUB识别启动分区，在那里加载配置信息
    6. GRUB为用户提供一个更改配置的机会
    7. 超时或者用户完成操作以后，GRUB执行配置
    8. 执行过程当中，GRUB可能会在启动分区中加载额外的代码
    9. GRUB执行boot命令，以加载和执行配置信息中linux命令指定的内核

系统配置

  1. 计算机硬件有一个使用电池的实时时钟（Real-time Clock，RTC），将硬件时钟设置为通用协调时间（Universal Coordinated Time，UTC），这样可以避免不同时区和夏令时带来的问题；
  1. 使用ntpdate（或者ntpd）从网络设置系统时间，然后使用硬件时钟命令进行同步；
  2. crontab：可以在任何时间运行任何程序（定时任务）；
  3. at：在将来的某一时刻一次性运行任务；

进程以及资源

  1. top命令将系统中最活跃的进程显示在最上方；
  2. lsof命令列出打开的文件以及使用它们的进程；
  3. strace（系统调用跟踪）和ltrace（系统库跟踪）命令能够帮助你了解程序试图执行哪些操作；
  4. 很多进程只有一个线程，叫单线程进程，有超过一个线程的叫多线程进程，多个线程可以同时在多个处理器上运行，ps和top命令添加-m选项可以看线程信息；
  5. uptime命令显示三个平均负载值和内核已经运行的时长；
  6. iostat在不带任何参数时显示系统当前的运行时间信息；
  7. iotop是为数不多的显示线程而非进程的工具；

总结

Linux上还有很多内容,比如shell的讲解需要单独的去记录,也算是对一本书里我认为重要的内容的记录!书名:《How Linux Works 2nd Edition》（《精通Linux》第二版 姜南 袁志鹏译），如果想获取更多的知识，可以自行阅读，可能收获的更多！
