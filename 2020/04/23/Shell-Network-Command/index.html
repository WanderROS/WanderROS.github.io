Shell Network Command

发表于 2020-04-23 | 分类于 Shell

简介

shell脚本可用于配置网络节点、测试主机是否可用、自动执行远程主机命令，而这些脚本的底层依赖是各种各样的网络命令，这部分学习的是常用的网络命令！

开始

基础网络知识

  1. 网络接口用于将主机以有线或无线的形式连接到网络；
  2. ifconfig命令用于配置及显示网络接口、子网掩码等细节信息；
  3. 命令示例：
  1. mac地址欺骗：
  1. 过滤出本机ip：

  1. host和nslookup是基于命令行的DNS查找工具都可以实现名字与IP地址的解析；

  2. ping是一个基础的网络命令，用于检验网络上主机之间的连通性，找出活动主机;

  3. ping命令如果执行顺利，会返回退出状态0；否则，返回非0;

  4. netstat命令用于显示各种网络相关信息，如网络连接、路由表、接口状态、masquerade连接、多播成员等等；

  5. route命令用于显示和操作路由表，要实现两个不同的子网之间的通信，需要一台连接两个网络的路由器，或者同时位于两个网络的网关来实现；

  6. traceroute命令可以显示分组途径的所有网关的地址,这些信息可以帮助我们搞明白分组到达目的地需要经过多少跳;

  7. curl命令是一个利用URL规则在命令行下工作的文件传输工具，支持文件的上传和下载，所以是综合传输工具。但按传统，习惯称curl为下载工具，作为一款强力工具，curl支持包括HTTP、HTTPS、FTP等众多协议，还支持POST、cookies、认证、从指定偏移处下载部分文件、用户代理字符串、限速、文件大小、进度条等特征；

  8. fping命令可以为多个IP地址生成ICMP分组，然后等待回应:

    1. -a指定显示出所有活动主机的IP地址;
    2. -u指定显示出所有不可达的主机;
    3. -g指定从“IP地址/子网掩码”记法或者“IP地址范围”记法中生成一组IP地址;

高级网络知识

  1. SSH代表的是Secure Shell（安全shell），使用加密隧道连接两台计算机，能够让你访问远程计算机上的shell；
  2. SSH执行指纹核对（fingerprint verification）来确保用户连接到正确的远程主机；
  3. 连接运行在端口422之上的SSH服务器: ssh user@locahost -p 422
  4. 要想在远程主机中执行命令，在本地shell中显示命令输出:ssh user@host ‘COMMANDS’,多条命令，命令之间用分号分隔:ssh user@host “command1 ; command2 ; command3”;
  5. 用ssh命令的选项-C启用对数据进行压缩传输;
  6. 可以使用ssh-copy-id命令将秘钥文件拷贝到要访问的服务器中，然后就可以免密登录了；
  7. 文件传输协议（File Transfer Protocol，FTP）是一个古老的协议，在很多公共站点上用于文件共享;
  8. SFTP是一个运行在SSH连接之上并模拟了FTP接口的文件传输系统;
  9. SCP是一个安全的文件复制命令，和旧式的、不安全的远程复制命令rcp类似,文件均通过SSH加密通道进行传输;
  1. scp的选项-p能够在复制文件的同时保留文件的权限和模式;
  2. sshfs利用SSH实现了在本地挂载点上挂载远程文件系统；

连接无线网络

  1. 无线网络需要使用iwconfig和iwlist工具来配置更多的参数；
  2. 无线LAN需要essid、key（密钥）以及frequency（频率）等参数；
    1. essid：无线网络的名称；
    2. key：一些网络需要用WEP密钥进行认证， WEP密钥通常是一个5位或10位十六 进制数口令；
    3. frequency：频率则是分配给特定网络的。
  3. iwlist工具能够扫描并列出可用的无线网络：iwlist scan；

分析网络流量与端口

  1. 通过获取开放端口列表、使用特定端口的应用以及运行该应用的用户，是跟踪系统中出现预期和非预期行为的一种方法，可用于检查rootkits或其他恶意软件；
  2. lsof 和 netstat 命令可用来列出端口以及运行在端口上的服务；
  3. lsof（list open files）命令可以列出已打开的文件。选项-i将范围限制在已打开的网络连接；
  4. netstat也可以显示网络服务统计信息;
  5. netcat或nc命令都可以创建用于在TCP/IP网络上传输数据的套接字;
  6. Linux系统可以作为网桥使用,网桥是一种低层连接，它并不是基于IP地址，而是使用MAC地址传递分组;

私有网络（VPN）

  1. 虚拟私有网络（Virtual Private Network，VPN）是建立在公网之上的加密通道；
  2. VPN可用于连接远程办公点、散布多处的生产制造站点以及远程工作人员!
  3. 后续专门讲解如何配置以及原理！
