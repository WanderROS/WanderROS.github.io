Linux 三剑客

发表于 2020-04-23 | 分类于 Linux

简介

shell脚本语言包含了众多用于解决Unix/Linux系统问题的工具，其中有不少和文本处理相关,包括sed、awk、grep和cut，这些工具可以相互结合以满足文本处理需求。这些命令的熟练掌握可以使得我们的Shell脚本编程达到新的层次。这些实用工具能够帮助我们在不同的层面上处理文本文件，比如字符、行、单词、行列等。正则表达式是一种基础的模式匹配技术。大多数文本处理工具都支持正则表达式。借助适合的正则表达式，我们可以对文本文件执行过滤、剥离（strip）、替换、搜索等操作。

开始

正则表达式

  1. 正则表达式是基于模式匹配的文本处理技术的关键所在，想要有效地运用正则表达式，就必须对其有一个基本的理解；
  2. [a-z0-9_]+ @[a-z0-9]+.[a-z]+.就是一个典型的能够匹配电子邮件地址的正则表达式；
  3. 正则表达式是由字面文本和具有特殊意义的符号组成的，可以根据需要，构造出适合的正则表达式来匹配任何文本；

正则表达式规则

  1. 位置标记锚点：标识字符串位置的正则表达式：
    1. ^：指定了匹配正则表达式的文本必须起始于字符串的首部；
    2. $：指定了匹配正则表达式的文本必须结束于目标字符串的尾部。
  2. 标识符：定义了那些为了匹配正则表达式，必须存在或不存在的字符：
    1. A字符：正则表达式必须匹配该字符；
    2. .：匹配任意一个字符；
    3. []：匹配括号内的任意一个字符，中括号内可以是一个字符组或字符范围；
    4. [^]：匹配不在括号中的任意一个字符，中括号内可以是一个字符组或字符范围。
  3. 数量修饰符：一个标识符可以出现一次、多次或是不出现：
    1. ?：匹配之前的项1次或0次；
    2. +：匹配之前的项一次或多次；
    3. *：匹配之前的项0次货多次；
    4. {n}：匹配之前的项n次；
    5. {n, }：之前的项至少匹配n次；
    6. {n,m}：之前的项必须匹配最小n次，最大m次，m>n。
  4. 其他匹配符：
    1. ()：将括号中的内容视为一个整体；
    2. |：指定了一种选择结构，可以匹配|两边的任意一项；
    3. \：转义字符可以转义之前介绍的特殊字符。
  5. 样例：
  1. grep命令默认使用基础正则表达式，选项-E可以使grep使用扩展正则表达式；
  2. 默认启用扩展正则表达式的egrep命令；
  3. 选项-o可以只输出匹配到的文本；
  4. 选项-v可以打印出不匹配match_pattern的所有行；
  5. 选项-c能够统计出匹配模式的文本行数，并不是匹配的次数；
  6. 如果需要在多级目录中对文本进行递归搜索，可以使用命令：grep “text” . -R -n；
  7. 选项-i可以在匹配模式时不考虑字符的大小写；
  8. 选项-e可以指定多个匹配模式；
  9. 选项-f可以读取文件并使用其中的模式（一个模式一行）；
  10. grep可以在搜索过程中使用通配符指定（include）或排除（exclude）某些文件；

cut命令切分文件

  1. cut命令可以按列，而不是按行来切分文件；
  2. cut命令能够提取指定位置或列之间的字符，可以指定每列的分隔符；
    1. 选项-f可以指定要提取的字段，由列号组成，彼此之间用逗号分隔：cut -f 2,3 filename；
    2. 选项-d能够设置分隔符；
    3. –complement选项显示出没有被-f指定的那些字段(mac上没有);
    4. 选项–output-delimiter可以指定输出分隔符(mac上没有)。

sed流编辑器

  1. sed是stream editor（流编辑器）的缩写，最常见的用法是进行文本替换；
  2. sed可以使用另一个字符串来替换匹配模式，模式可以是简单的字符串或正则表达式：sed ‘s/pattern/replace_string/‘ file；
  1. g标记可以使sed执行全局替换:sed ‘s/pattern/replace_string/g’ file;
  2. /#g标记可以使sed替换第N次出现的匹配;
  3. sed命令会将s之后的字符视为命令分隔符:
  1. 空行可以用正则表达式 ^$ 进行匹配,sed可以移除空行：
  1. 就地（in place）修改文件内容，可以使用选项-i：sed ‘s/PATTERN/replacement/‘ -i filename；
  2. 在sed中，我们可以用&指代模式所匹配到的字符串：
  1. 可以使用# 来指代出现在括号中的部分正则表达式所匹配到的内容：
  1. 可以利用管道组合多个sed命令，多个模式之间可以用分号分隔，或是使用选项-e PATTERN:
  1. sed表达式通常用单引号来引用,也可以使用双引号,shell会在调用sed前先扩展双引号中的内容;

高级文本处理awk

  1. awk命令可以处理数据流。它支持关联数组、递归函数、条件语句等功能；
  2. awk脚本的结构：
  1. awk脚本通常由3部分组成：BEGIN、END和带模式匹配选项的公共语句块,这3个部分都是可选的，可以不用出现在脚本中;
  2. 处理规则：awk以逐行的形式处理文件。BEGIN之后的命令会先于公共语句块执行。对于匹配PATTERN的行，awk会对其执行PATTERN之后的命令。最后，在处理完整个文件之后，awk会执行END之后的命令；
  3. 统计文件的行数：
  1. 模式本身可以是正则表达式、条件语句以及行范围等。如果当前行匹配该模式，则执行{ }中的语句。

awk的补充

  1. 特殊变量：
    1. NR：表示记录编号，当awk将行作为记录时，该变量相当于当前行号；
    2. NF：表示字段数量，在处理当前记录时，相当于字段数量，默认的字段分隔符是空格；
    3. $0：该变量包含当前记录的文本内容；
    4. $1：该变量包含第一个字段的文本内容；
    5. $2：该变量包含第二个字段的文本内容；
  2. 选项-v，我们可以将外部值（并非来自stdin）传递给awk;
  3. awk默认读取文件中的所有行,如果只想读取某一行，可以使用getline函数:
  1. 选项-F指定不同的分隔符，可以在BEGIN中使用 FS指定，在BEGIN语句块中可以用OFS=”delimiter”设置输出字段分隔符；
  1. 除了数字和字符串类型的变量，awk还支持关联数组,关联数组是一种使用字符串作为索引的数组,可以通过中括号中索引的形式来分辨出关联数组；
  2. 在awk中可以使用for循环，其格式与C语言中的差不多，awk还支持列表形式的for循环:
  1. awk内建的字符串处理函数:
    1. length(string)：返回字符串string的长度;
    2. index(string, search_string)：返回search_string在字符串string中出现的位置;
    3. split(string, array, delimiter)：以delimiter作为分隔符，分割字符串string，将生成的字符串存入数组array;
    4. substr(string, start-position, end-position) ： 返回字符串 string 中以start-position和end-position作为起止位置的子串;
    5. sub(regex, replacement_str, string)：将正则表达式regex匹配到的第一处内容替换成replacment_str;
    6. gsub(regex, replacement_str, string)：和sub()类似,不过该函数会替换正则表达式regex匹配到的所有内容;
    7. match(regex, string)：检查正则表达式regex是否能够在字符串string中找到匹配,如果能够找到，返回非0值；否则，返回0。match()有两个相关的特殊变量，分别是RSTART和RLENGTH。变量RSTART包含了匹配内容的起始位置，而变量RLENGTH包含了匹配内容的长度;

应用

  1. 统计词频(模式\b[[:alpha:]]+\b能够匹配每个单词并去除空白字符和标点符号)：
  1. 可以用paste命令实现按列合并；
  2. 打印从M行到N行之间的文本: awk ‘NR==M, NR==N’ filename;
  3. 能够匹配电子邮件地址的正则表达式: [A-Za-z0-9._]+@[A-Za-z0-9.]+.[a-zA-Z]{2,4}
  4. 匹配HTTP URL与匹配电子邮件地址类似，只是不需要匹配name@部分: http://[a-zA-Z0-9.]+.[a-zA-Z]{2,3}
