发表于 2020-04-22 | 分类于 Linux

开始

基本命令

  1. echo是用于终端打印的最基本命令，echo在每次调用后会添加一个换行符；
  2. printf命令也可用于终端打印，使用的参数和C语言中的printf函数一样；
  3. 注意：使用echo和printf的命令选项时，要确保选项出现在命令中的所有字符串之前；可以使用转义序列在终端中生成彩色文本;
  1. env或printenv命令查看当前shell 中所定义的全部环境变量;
  2. export命令声明了将由子进程所继承的一个或多个变量;
  3. Bash shell使用let、(( ))和[ ]执行基本的算术操作;
  4. sleep命令可以延迟脚本执行一段时间（以秒为单位）;
  5. 函数定义 function fname(){}
  6. pwd命令可以打印出工作目录的路径;
  7. cd命令可以将当前目录修改成指定的目录;
  8. read能够从键盘或标准输入中读取文本;
  9. 面向列表的for循环 for var in list; do commands; done

查找命令find

  1. find命令能够基于通配符或正则表达式、目录树深度、文件日期、文件类型等条件查找文件;
  2. -print选项使用\n（换行符）分隔输出的每个文件或目录名
  3. -name 选项指定了待查找文件名的模式；比如查找”.txt”能匹配所有名字以.txt结尾的文件或目录（单引号能够阻止shell扩展.txt）；
  1. -iname（忽略字母大小写），该选项的作用和-name类似，只不过在匹配名字时会忽略大小写；
  2. 支持逻辑操作符。-a和-and选项可以执行逻辑与（AND）操作，-o和-or选项可 以执行逻辑或（OR）操作；
  1. -path 选项可以限制所匹配文件的路径及名称;
  2. -regex选项和-path类似，只不过前者是基于正则表达式来匹配文件路径的;
  3. -iregex选项可以让正则表达式在匹配时忽略大小写;
  4. 用!排除匹配到的模式;
  1. -maxdepth和–mindepth选项可以限制find命令遍历的目录深度,可以避免出现死循环式的查找，-mindepth选项设置的是find开始进行查找的最小目录深度；
  2. -type选项对文件搜索进行过滤；
  3. 根据文件的时间戳进行搜索:
    1. 访问时间（-atime）：用户最近一次访问文件的时间;
    2. 修改时间（-mtime）：文件内容最后一次被修改的时间;
    3. 变化时间（-ctime）：文件元数据（例如权限或所有权）最后一次改变的时间;
    4. -atime、-mtime和-ctime可作为find的时间选项,可以用整数值来指定天数,这些数字前面可以加上-或+,-表示小于，+表示大于;
    5. -amin（访问时间）；
    6. -mmin（修改时间）；
    7. -cmin（变化时间）;
  4. –newer选项可以指定一个用于比较修改时间的参考文件，然后找出比参考文件更新的所有文件;
  5. 基于文件大小的搜索:
    1. b：块（512字节）;
    2. c：字节;
    3. w：字（2字节）;
    4. k：千字节（1024字节）;
    5. M：兆字节（1024K字节）;
    6. G：吉字节（1024M字节）;

  1. 基于文件权限和所有权的匹配:-perm选项指明find应该只匹配具有特定权限值的文件;

  2. -user USER就能够找出由某个特定用户所拥有的文件;

  3. 执行操作：

    1. -delete选项可以删除所匹配到的文件；
    2. -exec选项，find命令可以结合其他命令使用；
    3. find命令使用一对花括号{}代表文件名,对于每一个匹配的文件会将{}替换成相应的文件名,并进行相应的操作；
    4. 注意该命令结尾的;，必须对分号进行转义，否则shell会将其视为find命令的结束；

  4. find跳过特定的目录:

    1. 在find的执行过程中，跳过某些子目录能够提升性能;
    2. -prune选项排除某些符合条件的文件;

命令xargs

  1. xargs命令从stdin处读取一系列参数，然后使用这些参数来执行指定命令；
  2. 能将单行或多行输入文本转换成其他格式，例如单行变多行或是多行变单行；
  3. xargs命令应该紧跟在管道操作符之后；
  4. zargs命令重新格式化stdin接收到的数据，再将其作为参数提供给指定命令，xargs默认会执行echo命令,用来将多行输入转换成单行输出;
  5. xargs的-n选项可以限制每次调用命令时用到的参数个数;
  6. -d选项可以为输入数据指定自定义的分隔符;
  7. xargs命令可以同find命令很好地结合在一起,find的输出可以通过管道传给xargs，由后者执行-exec选项所无法处理的复杂操作;
  8. -I选项用于指定替换字符串，这个字符串会在xargs解析输入时被参数替换掉，{}会在每次执行中被替换为相应的参数；
  1. 需要注意find命令和xargs命令之间的联合使用的危险性；
  2. xargs -0表示xargs用NULL来作为分隔符;

命令tr

  1. 对来自标准输入的内容进行字符替换、字符删除以及重复字符压缩；
  2. 将输入中的字符由大写转换成小写：
  1. tr中利用集合的概念，可以轻松地将字符从一个集合映射到另一个集合中;
  2. tr命令可以用来加密,ROT13是一个著名的加密算法,在ROT13算法中，字符会被移动13 个位置，因此文本加密和解密都使用同一个函数;
  1. tr还可以将制表符转换成单个空格；
  2. -d选项可以通过指定需要被删除的字符集合将特定字符清除掉；
  3. -c选项来使用集合的补集,一般配合-d选项使用；
  4. tr命令能够完成很多文本处理任务，它可以删除字符串中重复出现的字符，比如空格等，使用-s选项；
  5. tr可以将不同的字符类作为集合使用(使用格式tr [:class:] [:class:]):
    1. alnum：字母和数字;
    2. alpha：字母;
    3. cntrl：控制（非打印）字符;
    4. digit：数字;
    5. graph：图形字符;
    6. lower：小写字母;
    7. print：可打印字符;
    8. punct：标点符号;
    9. space：空白字符;
    10. upper：大写字母;
    11. xdigit：十六进制字符

校验命令

  1. 校验和（checksum）程序用来从文件中生成相对较小的唯一密钥，绝大多数软件发行版都包含了一个校验和文件；
  2. 强健性最好且使用最为广泛的校验和算法是MD5和SHA-1；
  3. md5和shasum命令可以分别生成文件的md5值和sha值；
  4. 校验和有助于核实下载文件的完整性；
  5. md5deep或sha1deep命令可以遍历目录树，计算其中所有文件的校验和,-r选项允许md5deep递归遍历子目录,-l选项允许显示相对路径;

  1. ${VAR%.*} 的含义：
    1. 从 $VAR中删除位于%右侧的通配符（在上例中是.*）所匹配的字符串，通配符从右向左进行匹配；
    2. 给VAR赋值，即VAR=sample.jpg。通配符从右向左匹配到的内容是.jpg，因此从$VAR中删除匹配结果，得到输出sample；
    3. %属于非贪婪（non-greedy）操作，右向左找出匹配通配符的最短结果；
    4. %%，它与%相似，但行为模式却是贪婪的，这意味着它会匹配符合通配符的最长结果。
  2. #操作符可以从文件名中提取扩展名，这个操作符与%类似，不过求值方向是从左向右；
  3. ##从左向右进行贪婪匹配，并从指定变量中删除匹配结果；

交互式输入

  1. echo命令和重定向可以实现交互式输入的自动化；
  2. 要实施攻击，我们需要将十六进制形式的shellcode（例如\xeb\x1a\x5e\x31\xc0\x88\x46）进行重定向；
  3. expect程序能够执行复杂的交互操作并适应目标应用的变化,被用于控制硬件测试、验证软件构建、查询路由器统计信息等；
    1. spawn： 运行新的目标应用；
    2. expect： 关注目标应用发送的模式；
    3. send：向目标应用发送字符串
  4. spawn命令的参数是需要自动化运行的应用程序及其参数，expect命令接受一组模式以及匹配模式时要执行的操作，send命令是要发送的信息。
