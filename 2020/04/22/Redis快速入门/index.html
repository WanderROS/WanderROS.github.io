Redis快速入门

发表于 2020-04-22 | 分类于 Redis

简介

Redis在Web开发社区中是最火热的内存数据库之一，在初创企业乃至新浪微博这样的大公司都能看到Redis的使用，足以看到Redis的火热。Redis是一个开源的、高性能的、基于键值对的缓存与存储系统，通过提供多种键值数据类型来适应不同场景下的缓存与存储需求。Redis还可以胜任消息队列、任务队列等不同的角色。接下来开始学习Redis吧！

开始

基础介绍

  1. 字典（映射、关联数组）：一种数据结构，在字典中可以获取或设置键名对应的键值，也可以删除一个键值对；
  2. Redis是REmote DIctionary Server（远程字典服务器）的缩写，以字典结构存储数据，并允许其他应用通过TCP协议读写字典中的内容；
  3. 支持的键值数据类型：
    1. 字符串类型
    2. 散列类型
    3. 列表类型
    4. 集合类型
    5. 有序集合类型
  4. Redis字典结构的存储方式和对多种键值类型的支持使得开发者可以将程序中的数据直接映射到Redis中，数据在Redis中的存储形式和其在程序中的存储方式非常接近；
  5. 优势：Redis为不同数据类型的键值提供了非常方便地操作方式，比如集合类型提供了交集 、并集的集合运算操作；
  6. Redis数据库中的所有数据都存储在内存中，因此比较快！Redis防止内存中数据丢失，提供了持久化支持；
  7. Redis可以为每个键设置生存时间（Time to Live，TTL），生存时间到期后会自动删除键值对，可以充当缓存系统来使用；
  8. Redis的列表类型键可以用来实现队列，而且支持阻塞式读取，因而很容易实现一个高性能的优先队列；
  9. Redis提供了100多个命令，实际上常用的就十几个，因而使用相对简单；

使用

  1. Mac上安装:

  1. Redis相关的启动命令：

    1. redis-server：Redis服务器；
    2. redis-cli：Redis命令行客户端；
    3. redis-benchmark：Redis性能测试工具；
    4. redis-check-aof：AOF文件修复工具；
    5. redis-check-dump：RDB文件检查工具；
    6. redis-sentinel：Sentinel服务器；

  2. 启动Redis服务器使用命令redis-server就可以了，可以修改端口，添加 –port portnum ;

  3. 关闭redis：redis-cli SHUTDOWN ；

  4. redis-cli（Redis Command Line Interface）是Redis自带的基于命令行的Redis客户端,在这里主要用来学习Redis；

命令返回值

  1. 状态回复（status reply）：状态回复直接显示状态信息，比如PING 返回PONG；
  2. 错误回复（error reply）：错误回复以(error)开头，并在后面跟上错误信息；
  3. 整数回复（integer reply）:递增键值的INCR命令会以整数形式返回递增后的键值，还有一些其他命令如获取当前数据库中键的数量的DBSIZE命令，以（integer）开头，并在后面跟上整数数据；
  4. 字符串回复（bulk reply）：当请求一个字符串类型键的键值或者一个其他类型键中的某个元素时就会得到一个字符串回复，以双引号包裹；
  5. 多行字符串回复（multi-bulk reply）：请求一个非字符串类型键的元素列表时就会收到多行字符串回复，每行字符串都以一个序号开头；比如KEYS 命令可以获取数据库中符合指定规则的键名–KEYS *；

配置

  1. Redis支持通过配置文件来设置选项，可以通过启动参数传递同名的配置选项会覆盖配置文件中的相应参数；
  2. 在redis-cli中可以动态修改配置，如CONFIG SET loglevel warning ，也可以使用CONFIG GET命令获得Redis当前的配置情况：
  1. 一个Redis实例提供了多个用来存储数据的字典,客户端可以指定数据存储在哪个字典中,每个字典对外都是以一个从0开始递增数字命名,默认支持16个字典，可以通过配置参数databases来修改这一个数字，默认连接0号字典，可以随时使用SELECT命令更换数据库；
  1. 一个客户端要么可以访问全部数据库，要么就没有任何一个数据库的访问权限；
  2. redis的数据库之间不是完全隔离的，比如FLUSHALL命令可以清空一个Redis实例中所有数据库中的数据;
  3. 不同的应用应该使用不同的Redis实例存储数据；

Redis命令属性

  1. Redis不同命令拥有不同的属性，一个命令可以拥有多个属性；
  2. REDIS_CMD_WRITE：可以修改Redis数据库的数据；
  3. REDIS_CMD_DENYOOM：有可能增加Redis占用的存储空间，超过空间占用限制时，会拒绝执行拥有该属性的命令；
  4. REDIS_CMD_NOSCRIPT:拥有该属性的命令无法在Redis脚本中执行；
  5. REDIS_CMD_RANDOM：执行该属性的命令之后，就不能执行拥有REDIS_CMD_WRITE属性的命令；
  6. REDIS_CMD_SORT_FOR_SCRIPT：拥有该属性的命令会产生随机结果，在脚本中调用拥有该属性的命令会对结果进行排序；
  7. REDIS_CMD_LOADING：Redis启动时，会只执行拥有该属性的命令；

常见键值存储系统

  1. Voldemort
  2. Dynamo
  3. Memcachedb
  4. Cassandra
  5. Memcached
  6. Hypertable

为什么需要Redis

  1. 新的场景mysql的应用成了瓶颈：
    1. 线程阻塞，访问卡顿；
    2. List类型业务不适合mysql；
  2. 新的关系计算需求实现困难，大量的关系计算；

优化使用Redis

  1. 容量规划：
    1. 提前预估容量，上线前预拆足够的数据分片；
    2. 选择合适的数据类型，慎用zset；
    3. 业务独立存储，拒绝混放；
  2. 提高可用性：
    1. 所有Redis全部增加Slave；
    2. Master挂载slave不超过2个采用M-S-S方式挂载；
    3. 多IDC单Master,复制同步;
  3. 凌晨低峰升级,访问IP->域名：
    1. 出现问题——>错峰升级
    2. 及时清理pagecache，减少对正常业务的影响；
    3. AOF去掉rewrite，改用rotate；
    4. 支持热升级，避免重启，提高可运维性；
    5. 负载均衡；
  4. 资源服务化：
    1. configserver用于服务的发布与订阅；
    2. cacheService用于集群管理：
      1. 数据路由
      2. 负载均衡
      3. 数据在线迁移
      4. 服务治理
    3. 运维标准化、自动化

操作String类型

  1. SET key value：设置键值对；
  2. GET key: 读取键值；
  3. GETSET key value：重新设置键的键值，返回旧的键值；
  4. MGET key1 key2 … keyN：返回多个键值；
  5. SETNX key value：键值不存在时设置键值；
  6. SETEX key time value：超时自动删除键值；
  7. MSET key1 value1 key2 value2 … keyN valueN：设置多个键值对；
  8. MSETNX key1 value1 key2 value2 … keyN valueN：设置多个键值对，键值不存在时设置键值；
  9. INCR key：增加整数键值的值；
  10. INCRBY key integer：增加指定增长值；
  11. DECR key：减少整数键值的值；
  12. DECRBY key integer：减少整数键值的指定值；
  13. APPEND key value：追加指定键的键值；
  14. SUBSTR key start end：返回指定键的键值的子串；

操作列表类型

  1. RPUSH key value：在列表的右侧添加键值，可以多个；
  2. LPUSH key value：在列表的左侧添加键值，可以多个；
  3. LLEN key：返回指定列表类型键的长度；
  4. LRANGE key start end：返回列表指定范围的键值；
  5. LTRIM key start end：删除超出指定范围的数据；
  6. LINDEX key index：返回列表中指定位置的键值；
  7. LSET key index value：设置指定键的指定位置的键值；
  8. LREM key count value：从左侧删除匹配的元素多个；
  9. LPOP key：从列表左侧删除元素，并返回删除的键值；
  10. RPOP key：从列表右侧删除元素，并返回删除的键值；
  11. BLPOP key1 key2 … keyN timeout：超时左侧弹出键值；
  12. BRPOP key1 key2 … keyN timeout：超时右侧弹出键值；
  13. RPOPLPUSH srckey dstkey：从srckey的右侧弹出然后从左侧压入dstkey中；

操作集合类型

  1. SADD key member：添加指定的成员到指定的集合中；
  2. SREM key member：删除指定的成员到指定的集合中；
  3. SPOP key [count]：从指定集合中随机删除指定个键值；
  4. SMOVE srckey dstkey member：转移指定键值到指定的键中；
  5. SCARD key：返回指定集合的元素数量；
  6. SISMEMBER key member：检查指定元素是否是集合中的元素；
  7. SINTER key1 key2 … keyN：返回集合的交集；
  8. SINTERSTORE dstkey key1 key2 … keyN：将集合之间的交集保存到dstkey中；
  9. SUNION key1 key2 … keyN：返回集合的并集；
  10. SUNIONSTORE dstkey key1 … keyN：将集合之间的并集保存到dstkey中；
  11. SDIFF key1 key2 … keyN：返回集合的差集；
  12. SDIFFSTORE dstkey key1 … keyN：将集合之间的差集保存到dstkey中；
  13. SMEMBERS key：返回集合的所有元素；
  14. SRANDMEMBER key：随机返回集合中的一个元素；

操作有序集合类型

  1. ZADD key score member：通过给定的分数插入成员值到指定的key的有序集合中，存在则插入失败；
  2. ZREM key member：从给定的key中删除指定的成员；
  3. ZINCRBY key increment member：给指定的key的成员积分递增指定值；
  4. ZRANK key member：返回指定有序集合中的成员的排名，从0开始计数；
  5. ZREVRANK key member：返回指定有序集合中的成员的排名，从0开始计数，倒序；
  6. ZRANGE key start end：通过开始值和结束值来取有序集合中的元素；
  7. ZREVRANGE key start end：通过开始值和结束值来取有序集合中的元素，逆序；
  8. ZRANGEBYSCORE key min max：通过分数范围返回指定有序集合中的元素；
  9. ZCOUNT key min max：返回指定分数范围的有序集合中的元素数目；
  10. ZCARD key：获取指定有序集合的成员基数；
  11. ZSCORE key element：返回指定集合的元素的分数；
  12. ZREMRANGEBYRANK key min max:删除指定的有序集合中指定排名范围的成员；
  13. ZREMRANGEBYSCORE key min max:删除指定的有序集合中指定分数范围的成员；
  14. zinterstore/zunionstore storeKey keyNum key [key …] [weights weight [weight…]] [aggregate sum|min|max]　:通过指定的keys做交集或者并集并保存到指定的集合中；

操作散列类型

  1. HSET key field value：设置hash的字段及值；
  2. HGET key field：获取hash的指定字段的值；
  3. HMGET key field1 … fieldN：返回指定的hash的多个字段的值；
  4. HMSET key field1 value1 … fieldN valueN：插入指定的hash多个字段及值；
  5. HINCRBY key field integer：对指定的hash字段的值进行递增操作
  6. HEXISTS key field：检查指定的hash字段是否存在；
  7. HDEL key field：通过指定的hash字段删除hash值；
  8. HLEN key：返回hash的长度；
  9. HKEYS key：返回所有hash的所有keys；
  10. HVALS key：返回指定hash的所有value；
  11. HGETALL key：返回指定的hash所有字段及关联的值；

键值相关命令

  1. keys pattern：返回满足给定模式的所有key；
  2. exists key：确定一个key是否存在；
  3. del key：删除一个key；
  4. expire key time：设置一个key的过期时间（单位：秒）；
  5. move：将当前数据库中的key转移到其他数据库中；
  6. persist key：移除给定key的过期时间；
  7. randomkey：随机返回一个key；
  8. rename：重命名key；
  9. type key：返回key的类型；
  10. ttl key：显示key的超期时间；
  11. flushall：清除所有的key；

总结

Redis真的用的非常广泛，还需要学习如何在自己的项目中使用Redis，这部分主要介绍一下基础的Redis知识。
